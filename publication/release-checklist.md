# Release Checklist

## Manuscript

- [ ] Thesis is narrow and defensible.
- [ ] Private entity data removed.
- [ ] Every comparative claim has evidence.
- [ ] No quantitative superiority claim without benchmark data.
- [ ] Limitations section is explicit.

## References

- [ ] Every citation resolves.
- [ ] Every source has type and access date.
- [ ] Vendor claims are marked as vendor/product documentation.
- [ ] No search-result snippets cited.
- [ ] No fabricated references.

## Data

- [ ] Only synthetic or sanitized data included.
- [ ] JSON validates.
- [ ] Hashes present for fixtures/artifacts.
- [ ] No secrets or credentials.

## Repository

- [ ] License approved.
- [ ] CITATION metadata approved.
- [ ] README public-safe.
- [ ] No assistant/model/bot byline.
- [ ] GitHub repo name and visibility approved.

## Publishing

- [ ] Ahmed approves final files.
- [ ] Ahmed approves commit.
- [ ] Ahmed approves push.
- [ ] Ahmed approves GitHub release.
- [ ] Ahmed approves PDF build/distribution.
